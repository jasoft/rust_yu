## Why

当前安装软件列表信息过于基础，缺少图标、安装日期与占用空间等关键字段，导致用户难以快速识别与评估待卸载软件。参考 `UninstallerWPF` 的后端思路补齐这些元数据，并引入缓存机制，可以在不牺牲安全性的前提下明显提升列表可读性与扫描响应速度。

## What Changes

- 在已安装软件扫描结果中新增展示字段：应用图标、安装日期、占用空间（可用时提供）。
- 增加元数据聚合流程：结合注册表与文件系统信息，生成统一的软件展示模型。
- 增加扫描结果缓存：支持读取缓存、失效重建与主动刷新，减少重复全量扫描。
- 为缺失或低置信度字段提供明确标记，避免误导性展示。
- 对前后端命令返回结构进行扩展，使前端可直接渲染新增字段与缓存状态。

## Capabilities

### New Capabilities
- `installed-app-metadata`: 定义并提供已安装应用的增强元数据（图标、安装日期、占用空间、字段置信度/可用性）。
- `installed-app-scan-cache`: 定义扫描结果缓存的生命周期与一致性行为（命中、失效、重建、强制刷新）。

### Modified Capabilities
- 无

## Impact

- 受影响代码：
  - Rust 后端扫描与列表命令（注册表读取、文件系统探测、结果组装）。
  - Tauri command 返回类型与序列化结构。
  - 前端应用列表视图与字段展示逻辑。
- 依赖与系统影响：
  - 可能新增/调整图标提取与目录大小计算相关 crate 或 Windows API 调用。
  - 需要本地缓存存储（如应用数据目录下 JSON/SQLite）及失效策略。
- 风险与约束：
  - 占用空间计算属于 I/O 密集型，需异步执行并避免阻塞 UI。
  - 对不确定归属的文件/目录保持保守策略，仅展示或标记低置信度，不做激进推断。
